ZyntraPay - Intacct-Style API Reference

Framework: .NET 8 Web API
Architecture: Microservices + Ocelot
Gateway: http://localhost:5000/gateway
Auth: JWT Bearer

This version uses endpoint cards with request schema, response examples, and consistent auth/status guidance.

Common Request Headers

Content-Type: application/json
Authorization: Bearer {token} // for protected endpoints

Auth Service

POST/gateway/auth/register

Auth: Public

Request Fields
FieldTypeRequired
emailstringYes
phoneNumberstringYes
passwordstringYes
{
  "email": "john.doe@example.com",
  "phoneNumber": "9876543210",
  "password": "SecurePass123!"
}
Success Response (200) 
{
  "message": "User registered successfully"
}

POST/gateway/auth/login

Auth: Public

Request 
{
  "email": "john.doe@example.com",
  "password": "SecurePass123!"
}
Success Response (200) 
{
  "token": "jwt-token",
  "refreshToken": "refresh-token",
  "email": "john.doe@example.com",
  "role": "User",
  "phoneUpdateRequired": false
}

PUT/gateway/auth/update-phone

Auth: User

Request 
{
  "phoneNumber": "9123456789"
}
Success Response (200) 
{
  "message": "Phone number updated successfully"
}
